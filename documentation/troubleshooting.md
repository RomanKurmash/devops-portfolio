# Інструкція з усунення несправностей (Troubleshooting Guide)

Цей посібник містить практичні кроки для діагностики та розв'язання потенційних проблем в роботі моніторингового та SecOps стеку.

---

## 1. Діагностика локальної LLM (Ollama & GPU)

Якщо виявлення аномалій або аналіз причин відмовляє в роботі, перевірте стан Ollama:

### А. Перевірка доступності сервісу Ollama
Виконайте запит до API з хост-машини:
```bash
curl http://localhost:11434/api/tags
```
У відповіді має повернутися список завантажених моделей, включаючи `llama3`.

### Б. Моніторинг використання GPU
Перевірте, чи модель виконується на GPU (апаратне прискорення):
```bash
nvidia-smi
```
Під час активного інференсу (аналіз логів) ви повинні бачити процес `ollama` у списку запущених на GPU та підвищення використання відеопам'яті (VRAM).

---

## 2. Діагностика збору логів (Loki & Promtail)

Якщо логи не відображаються у Grafana або не аналізуються AI-адаптером:

### А. Перевірка статусів контейнерів
Перевірте, чи запущено Promtail та Loki:
```bash
docker ps -f name=promtail -f name=loki
```

### Б. Перевірка логів Promtail
Якщо логи не доставляються до Loki, перевірте помилки відправки:
```bash
docker logs promtail
```
Шукайте помилки з кодом `400` або `500` при спробі пушу логів (Connection refused).

## 3. Діагностика оповіщень у Telegram

Якщо сповіщення про алерти не надходять у месенджер:

### А. Перевірка логів Telegram-бота
```bash
docker logs telegram-bot
```
Перевірте наявність помилок ініціалізації:
* `Unauthorized` — некоректний `TELEGRAM_TOKEN`.
* `Chat not found` — бот не ініційований у чаті з користувачем (потрібно надіслати `/start` боту).

### Б. Перевірка зв'язку Alertmanager -> Telegram Bot
Перевірте, чи отримує Telegram-бот вебхуки від Alertmanager:
```bash
docker logs telegram-bot
```
У логах має відображатися успішне отримання POST-запиту:
`POST /webhook HTTP/1.1 200 OK`
